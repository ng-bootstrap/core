Accordion is a component that allows you to organize sections of the user interface into collapsable panels.

In the most primitive form, an accordion is used by creating an ngb-accordion element, with nested ngb-panel elements within it.

The above example also uses the ngbPanelContent directive to denote the container of the content to be displayed in the panel.

Some additional notes around basic usage of the component are the following:

  • The accordion element can be given a template reference variable, which allows external controls to drive the operation of the accordion (i.e., which panels are open).
  • Each ngb-panel can be given an id attribute, which would allow for direct access to identify or toggle the open status of the given panel.
  • It is also possible to assign the activeIds attribute of the ngb-accordion element to the value of one of the specified ngb-panel element's id to define which panel is open by default.
  • A panel can also be disabled, which prevents it from being able to be opened by using the disabled attribute.
  • The accordion itself can also be set to only allow one open panel at a time by using the closeOthers attribute.

Customization Info

There are really 2 ways to customize the usage of the accordion.

  • Custom Panel Headers
  • Custom Global Configuration

Custom Panel Headers

ngb-panel supports the title attribute for defining a simple, click-able header with text for the panel which works in many cases.

However, sometimes it is useful to be able to do something more advanced with the header, such as display custom HTML, or use a button or a specific click-able element to trigger the toggle behavior for the panel.

To accomplish this, the following directives will be helpful:

  1. ngbPanelHeader
  2. ngbPanelToggle
